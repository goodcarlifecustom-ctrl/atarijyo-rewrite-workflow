# 装飾プロンプト

リライト後の記事HTMLに対して、以下の装飾を行ってください。
WordPressテーマはSWELLを想定しています。

## 0. 「この記事でわかること」リストの設置

### 対象

記事全体の最初のH2見出しの直前。

### 内容

最初のH2見出しの直前に、以下を設置してください。

* 「【この記事でわかること】」というテキスト
* 記事内のすべてのH2見出しへのアンカーリンク付きリスト

### HTML形式

```html
<p><strong>【この記事でわかること】</strong></p>
<ul>
  <li><a href="#h2-title-1">H2見出しテキスト</a></li>
  <li><a href="#h2-title-2">H2見出しテキスト</a></li>
  <li><a href="#h2-title-3">H2見出しテキスト</a></li>
</ul>
```

### id属性

リスト内のリンク先となる各H2見出しタグには、アンカーリンクに対応するid属性を付与してください。

例：

```html
<h2 id="h2-title-1">H2見出しテキスト</h2>
```

既存のid属性がある場合は、原則として既存idを利用してください。
既存idがない場合は、内容が分かる自然なidを付与してください。

例：

```html
<h2 id="sapporo-app">札幌でセフレを探すなら出会い系アプリがおすすめ</h2>
```

idは重複しないようにしてください。

## 1. H3見出しへのアンカーリンク付きリストの設置

### 対象

H2配下に複数のH3見出しがあり、読者が先に内容を把握した方が読みやすい箇所。

特に以下のようなH2では設置を検討してください。

* 方法を複数紹介するH2
* 比較項目が並ぶH2
* 手順を説明するH2
* 注意点を複数紹介するH2
* おすすめサービスや選び方を紹介するH2

### 内容

対象H2の導入文、つまりH2直下の`<p>`タグの直後に、その下に続くH3見出しへのアンカーリンク付きリストを設置してください。

### HTML形式

```html
<p>導入文テキスト</p>
<ul>
  <li><a href="#heading-d">H3見出しD</a></li>
  <li><a href="#heading-e">H3見出しE</a></li>
  <li><a href="#heading-f">H3見出しF</a></li>
</ul>
```

### id属性

各H3見出しタグには、アンカーリンクに対応するid属性を付与してください。

例：

```html
<h3 id="heading-d">H3見出しD</h3>
```

既存のid属性がある場合は、それを利用してください。
新しくidを付与する場合は、見出し内容に合う分かりやすいidにしてください。

## 2. リストのボックス装飾

### 対象

以下のリストを優先してボックス化してください。

* 「この記事でわかること」のリスト
* H3見出しへのアンカーリンク付きリスト
* 読者の判断に役立つ重要な箇条書き
* メリット・デメリット
* 注意点
* 手順
* 比較項目

記事内のすべての`<ul>`を機械的にボックス化する必要はありません。
通常の短い箇条書きや、本文中の軽い補足リストはそのままでも構いません。

### 使用するHTML

リストを以下のSWELL用HTMLで囲んでください。
リスト本体の`<ul>...</ul>`は、必ず`div.cap_box_content`の中に配置してください。

```html
<!-- wp:loos/cap-block {"className":"is-style-onborder_ttl"} -->
<div class="swell-block-capbox cap_box is-style-onborder_ttl">
  <div class="cap_box_ttl"><span>{ここにタイトル}</span></div>
  <div class="cap_box_content">
    <ul>
      <li>リスト項目</li>
      <li>リスト項目</li>
    </ul>
  </div>
</div>
<!-- /wp:loos/cap-block -->
```

### タイトルの付け方

リスト内容に応じて、自然なタイトルを付けてください。

例：

* この記事でわかること
* まず確認したいポイント
* おすすめの選び方
* 注意点
* 比較ポイント
* 手順
* メリット
* デメリット

H2配下のH3アンカーリストの場合は、H2の内容に合わせてタイトルを付けてください。

例：

```html
<div class="cap_box_ttl"><span>この章で紹介する内容</span></div>
```

## 3. 本文の段落調整

### 対象

各H2、H3見出しに対応する本文。
特に、1つの`<p>`タグに長い文章がまとまっている箇所。

### 内容

各見出し下の本文が長く、1つの`<p>`タグにまとめられている場合は、読みにくさを解消するために、意味のまとまりや話題の転換点を考慮して、適切に複数の`<p>`タグへ分割してください。

### 分割の目安

以下のような単位で区切れるか確認してください。

* 導入
* 詳細説明
* 具体例
* 注意点
* 結論
* 補足

### 注意点

* 文章の意味や流れを変えないでください。
* 極端に短い段落にしないでください。
* 原則として、1つの`<p>`は2〜3文程度のまとまりを意識してください。
* この段落調整は、マーカー適用前に行ってください。

---

## 4. マーカーの適用

### 対象

段落調整後の各H2、H3見出しに対応する本文。
各`<p>`タグ内の重要箇所に適用してください。

### 適用ルール

各見出し下の本文全体を通して、最も重要と思われる箇所に原則1箇所だけマーカーを適用してください。

ただし、ポジティブな重要箇所とネガティブな重要箇所が明確に両方存在する場合は、それぞれにマーカーを適用しても構いません。
その場合も、1見出しにつき最大2箇所までにしてください。

### ポジティブな内容

メリット、良い点、おすすめ、結論、重要な訴求などには、以下の黄色マーカーを使ってください。

```html
<span class="swl-marker mark_yellow">強調する文章</span>
```

### ネガティブな内容

デメリット、注意点、課題、リスク、避けるべき行動などには、以下の装飾を使ってください。

```html
<mark style="background-color:rgba(0, 0, 0, 0)" class="has-inline-color has-swl-deep-01-color">強調する文章</mark>
```

### 注意点

* 既存のマーカーがある場合は、一度すべて削除してから適用し直してください。
* マーカーは文章の意味を強調する箇所にだけ使ってください。
* 1文全体に長くマーカーを引きすぎないでください。
* キーワードを目立たせる目的だけでマーカーを使わないでください。
* 読者にとって重要な判断材料になる箇所を優先してください。

---

## 5. 装飾時の禁止事項

以下は絶対に行わないでください。

* 既存のSWELLブロックを壊さない
* ショートコードを削除・変更しない
* アフィリエイトリンクを削除・変更しない
* 広告タグを削除・変更しない
* 計測タグを削除・変更しない
* 既存の画像を削除しない
* 既存の表を壊さない
* 見出しの意味を変えない
* 本文の事実関係を変えない
* 装飾を過剰に入れすぎない
* すべてのリストを機械的にボックス化しない

---

## 6. 出力時の注意

確認用HTMLである`rewrite_review.html`には、変更箇所の色付けを入れてください。

WordPress下書き投入用HTMLである`rewritten.html`には、確認用の色付けタグや凡例CSSを入れないでください。

ただし、SWELL装飾として必要な以下は残してください。

* アンカーリンク
* id属性
* SWELLのcap-box
* マーカー
* ボックス
* リスト
* 吹き出し
* 内部リンク
