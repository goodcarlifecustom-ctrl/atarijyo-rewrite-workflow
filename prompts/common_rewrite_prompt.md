# 共通リライトプロンプト

あなたはWordPress記事のSEOリライトを行うエンジニア兼編集者です。

このタスクでは、1記事のみを対象にリライトしてください。
複数記事を一括処理しないでください。

## 目的

Search Consoleデータ、Ahrefsデータ、現在の記事HTMLをもとに、SEO改善を目的とした簡易リライトを行ってください。

30記事を順番に処理する前提のため、1記事あたりの作業は「簡易リライト」に留めます。

## 重要

この記事HTMLは、Codexのタスク詳細欄に貼り付けられています。
zip内に `article.html` はありません。
このタスクに貼られた【現在の記事HTML】を対象記事の本文HTMLとして扱ってください。

## 作業手順

1. 現在の記事HTMLを解析してください。
   - title
   - h1
   - h2
   - h3
   - 本文
   - 既存の内部リンク
   - 既存の外部リンク
   - アフィリエイトリンク
   - SWELLブロック
   - ショートコード
   - 広告タグ
   - 画像
   - 表

2. Search Consoleデータを確認してください。
   以下のKWを優先してください。
   - クリック数が多いKW
   - 表示回数が多いKW
   - 平均掲載順位が5〜15位のKW
   - CTRが低く改善余地があるKW
   - title / h2 / h3 に入っていないKW

3. Ahrefsデータがある場合は補助的に使ってください。
   Search ConsoleデータとAhrefsデータを照合し、追加候補KWを分類してください。

4. KW分類は以下にしてください。
   - A：見出しに入れる
   - B：本文に自然に入れる
   - C：注意喚起・補足として扱う
   - D：使わない

5. 簡易リライトを実行してください。
   対象範囲は以下です。
   - タイトル案の改善
   - 導入文の改善
   - 不足KWを本文に自然追加
   - 必要なH2/H3を1〜3個追加
   - 古い表現の修正
   - 読みにくい文章の整理
   - 内部リンクの追加
   - 公的・権威性リンクの追加
   - アフィリエイト導線の自然な改善
   - 注意点の追記
   - まとめ文の改善

6. 競合サイトの見出しや本文をそのままコピーしないでください。
   検索意図を参考にする場合でも、独自の表現にしてください。

7. 既存の以下は必ず保持してください。
   - SWELLブロック
   - ショートコード
   - アフィリエイトリンク
   - 広告タグ
   - 計測タグ
   - 既存の表
   - 既存の画像
   - 重要な内部リンク

8. 装飾は `prompts/decoration_prompt.md` に従ってください。

9. WordPress投稿を行う場合は、`prompts/wp_safety_rules.md` を必ず守ってください。
